Preparei abaixo uma lista de links e materiais para que vocês possam ir lendo e se familiarizando com a temática de mapeamento das nossas áreas de interesse. A lista cobre desde os bancos de dados que já existem até o uso do Google Earth Engine (GEE), tanto em JavaScript quanto em Python, além de outros tópicos que serão bem relevantes para nós neste momento.

1. [**Catálogo de Datasets do GEE**](https://developers.google.com/earth-engine/datasets)**:** Aqui está o repositório central de imagens de satélite disponíveis no GEE. Vocês vão encontrar desde dados brutos para mapeamentos e classificações até coleções prontas de dados climáticos e mapas de uso e cobertura do solo.  
2. [**Dynamic World (DW)**](https://developers.google.com/earth-engine/datasets/catalog/GOOGLE_DYNAMICWORLD_V1)**:** Um dataset excelente que mapeia as alterações do uso e cobertura do solo com altíssima frequência (quase semanalmente), ótimo para análises temporais. *(Para explorar visualmente a plataforma deles, recomendo também o site [dynamicworld.app](https://dynamicworld.app/)).*  
3. [**Google Open Buildings (Footprints)**](https://sites.research.google/open-buildings/)**:** Dataset gerado pela Google com o mapeamento e delimitação de polígonos de construções (casas e prédios). Essencial para análises de adensamento e infraestrutura. *(Link direto para carregar o [dataset dentro do GEE](https://developers.google.com/earth-engine/datasets/catalog/GOOGLE_Research_open-buildings_v3_polygons)).*  
4. [**Introdução ao GEE em JavaScript (Code Editor)**](https://developers.google.com/earth-engine/guides/playground)**:** O guia oficial de como começar a utilizar o GEE diretamente no navegador através do Code Editor. É o melhor ambiente para prototipar códigos, testar lógicas e visualizar mapas rapidamente.  
5. [**Introdução ao GEE em Python**](https://developers.google.com/earth-engine/guides/python_install)**:** Documentação mostrando como instalar, autenticar e inicializar a API Python do GEE. É o caminho ideal para quem vai trabalhar em ambientes como o Google Colab ou rodar scripts locais pelo VS Code.

Fiquem à vontade para explorar os materiais. Se surgirem dúvidas ou se precisarem de ajuda para rodar os primeiros scripts, me avisem e marcamos um papo.
